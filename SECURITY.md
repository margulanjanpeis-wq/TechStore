## Қауіпсіздік осалдығын хабарлау

Жобада қауіпсіздік осалдығын тапсаңыз, төмендегі нұсқауларды орындаңыз:

### Хабарлау процесі

1. **Публикалық хабарламаңыз**: GitHub Issues-те публикалық хабарламаңыз
2. **Жеке хабарлаңыз**: Электрондық пошта арқылы хабарлаңыз
3. **Деректерді қамтыңыз**:
   - Осалдықтың сипаттамасы
   - Қайталау қадамдары
   - Әсер ету деңгейі
   - Мүмкін шешімдер

### Жауап беру уақыты

- Алғашқы жауап: 48 сағат ішінде
- Бағалау: 7 күн ішінде
- Түзету: Осалдықтың деңгейіне байланысты

## Қауіпсіздік шаралары

### Авторизация

- JWT токендер қолданылады
- Токен мерзімі: 30 минут
- Bcrypt пароль хэштеу (cost factor: 12)
- Пароль талаптары:
  - Минимум 8 символ
  - Әріптер мен сандар

### Database қауіпсіздігі

- Параметрленген сұраулар (SQL injection қорғанысы)
- Database пайдаланушысының шектеулі құқықтары
- Автоматты backup жүйесі
- Деректер шифрлау (өндірісте)

### API қауіпсіздігі

- CORS конфигурациясы
- Rate limiting (болашақта)
- Input validation
- Output sanitization
- HTTPS міндетті (өндірісте)

### Frontend қауіпсіздігі

- XSS қорғанысы
- CSRF токендер
- Secure cookies
- Content Security Policy

### Infrastructure қауіпсіздігі

- Docker контейнерлер изоляциясы
- Network segmentation
- Firewall конфигурациясы
- Логтарды мониторинг
- Автоматты жаңартулар
